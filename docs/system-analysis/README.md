# System Analysis Documentation

This folder contains the complete System Analyst documentation set for the BYOD Registration and Monitoring System.

## Document Index

| File | Purpose |
| --- | --- |
| `01-system-analysis.md` | Full system analysis: purpose, scope, users, workflows, modules, assumptions, risks, and recommendations. |
| `02-functional-requirements.md` | Numbered functional requirements grouped by module. |
| `03-non-functional-requirements.md` | Numbered quality requirements for usability, security, reliability, performance, maintainability, data integrity, and auditability. |
| `04-use-cases.md` | Actor list and detailed use case descriptions with flows, alternatives, exceptions, and postconditions. |
| `05-business-rules-and-validation-rules.md` | Business rules separated from enforceable system validation rules. |
| `06-data-requirements-and-data-dictionary.md` | Proposed database tables, field definitions, status values, constraints, and ERD draft. |
| `07-screen-requirements.md` | Screen-level requirements for UI/UX and JavaFX frontend development. |
| `08-report-requirements.md` | Report definitions, users, filters, columns, and output formats. |
| `09-acceptance-criteria-and-test-scenarios.md` | Acceptance criteria and QA scenarios using Given/When/Then where useful. |
| `10-gap-analysis-and-recommendations.md` | Review findings, fixed gaps, remaining questions, and future recommendations. |
| `11-requirements-traceability-matrix.md` | Traceability between business rules, functional requirements, screens, database tables, and tests. |
| `12-change-log.md` | Documentation creation log and major improvements. |
| `13-user-interactions.md` | Role-based guide showing what admins, guards, and students can interact with in the system. |
