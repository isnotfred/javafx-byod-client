# BYOD Device Management System — Installation & Packaging Guide

This guide provides the system requirements and detailed instructions for compiling and packaging the **BYOD Device Management Client** into native installers using Java's `jpackage` tool. It also details the step-by-step installation process for end users on different operating systems.

---

## 1. System Requirements

Because the native installer generated by `jpackage` bundles a self-contained, customized Java Runtime Environment (JRE), **end users do not need to install Java or configure any environment variables** on their machines.

### 1.1 End-User Client Requirements

| Category | Minimum Specification | Recommended Specification |
| :--- | :--- | :--- |
| **Operating System** | • **Windows**: Windows 10 or 11 (64-bit)<br>• **macOS**: macOS 11 (Big Sur) or newer (Intel or Apple Silicon)<br>• **Linux**: Ubuntu 20.04+, Debian 10+, Fedora 33+, or equivalent | • **Windows**: Windows 11 (64-bit)<br>• **macOS**: macOS 13 (Ventura) or newer<br>• **Linux**: Ubuntu 22.04 LTS or newer |
| **Processor** | Dual-core 2.0 GHz or faster | Quad-core 2.4 GHz or faster |
| **Memory (RAM)** | 4 GB RAM | 8 GB RAM or higher |
| **Storage Space** | ~150 MB free disk space (app + runtime bundle) | 500 MB free disk space |
| **Network** | Broadband internet connection for API connectivity | High-speed Wi-Fi or Ethernet connection |
| **Display** | 1280 × 720 resolution (High-DPI supported) | 1920 × 1080 resolution |

### 1.2 Network & Security Configuration

The JavaFX client communicates with the remote Spring Boot backend via HTTPS.
* **API Endpoint**: `https://java-byod-backend-production.up.railway.app` (Default)
* **Firewall Rule**: Allow outbound traffic on port `443` (HTTPS) to the API domain.
* **Security Protocol**: TLS 1.2 or TLS 1.3.

## 2. Developer Packaging Guide (Using `jpackage`)

To build a native installer, you must run the packaging tool on the target platform (e.g., compile on Windows to produce a Windows `.exe` or `.msi` installer).

### 2.1 Developer Prerequisites

Ensure you have the following installed on your packaging machine:
1. **JDK 17 or 21**: Must include `jlink` and `jpackage` utilities.
2. **Maven 3.8+**: To manage dependencies and compile the source code.
3. **OS-Specific Packaging Tools**:
   * **Windows**: [WiX Toolset v3.x](https://wixtoolset.org/releases/) (Required by `jpackage` to build `.msi` or `.exe` installers). Ensure the WiX bin folder is added to your system `PATH`.
   * **macOS**: Xcode Command Line Tools.
   * **Linux**: `rpm-build` (for `.rpm` packages) or `dpkg-dev` (for `.deb` packages).

### 2.2 Phase 1: Compile and Prepare Dependencies

Since `jpackage` builds the app from modular JAR files, we must compile the project and gather all dependency JARs (like Jackson modules) into a single folder.

Run the following commands in the project root:

```bash
# 1. Clean and package the Maven project
mvn clean package

# 2. Create a directory to store all runtime JAR dependencies
mkdir target/libs

# 3. Copy dependencies into target/libs
mvn dependency:copy-dependencies -DoutputDirectory=target/libs

# 4. Copy our compiled client JAR into target/libs as well
copy target\javafx-byod-client-1.0-SNAPSHOT.jar target\libs\
```
*(On Linux/macOS, use `cp target/javafx-byod-client-1.0-SNAPSHOT.jar target/libs/` instead of `copy`)*

### 2.3 Phase 2: Build the Installer

Execute `jpackage` targeting the compiled application module and its dependencies.

> [!NOTE]
> Below are the recommended `jpackage` configurations for each platform. You can supply an optional custom icon using the `--icon` parameter (e.g., a `.ico` file for Windows, `.icns` for macOS, or `.png` for Linux).

#### Windows (Produces `.exe` or `.msi`)

To build an EXE installer that installs the app, creates a Start Menu entry, and adds a desktop shortcut:

```cmd
jpackage `
  --name "BYODClient" `
  --app-version "1.0.0" `
  --description "BYOD Device Management Client" `
  --vendor "PUP BYOD Team" `
  --type exe `
  --input target/libs `
  --main-jar javafx-byod-client-1.0-SNAPSHOT.jar `
  --main-class com.pup.byod.javafxbyodclient.Launcher `
  --module-path target/libs `
  --module com.pup.byod.javafxbyodclient/com.pup.byod.javafxbyodclient.BYODApplication `
  --dest target/dist `
  --win-shortcut `
  --win-menu `
  --win-menu-group "BYOD System" `
  --win-dir-chooser
```

> [!TIP]
> To compile a silent or enterprise-ready `.msi` package instead of an `.exe`, simply change `--type exe` to `--type msi`.

#### macOS (Produces `.dmg` or `.pkg`)

To build a drag-and-drop DMG installer on macOS:

```bash
jpackage \
  --name "BYODClient" \
  --app-version "1.0.0" \
  --description "BYOD Device Management Client" \
  --vendor "PUP BYOD Team" \
  --type dmg \
  --input target/libs \
  --main-jar javafx-byod-client-1.0-SNAPSHOT.jar \
  --main-class com.pup.byod.javafxbyodclient.Launcher \
  --module-path target/libs \
  --module com.pup.byod.javafxbyodclient/com.pup.byod.javafxbyodclient.BYODApplication \
  --dest target/dist \
  --mac-package-name "BYOD Device Management"
```

#### Linux (Produces `.deb` or `.rpm`)

To build a Debian package for Ubuntu/Debian:

```bash
jpackage \
  --name "byod-client" \
  --app-version "1.0.0" \
  --description "BYOD Device Management Client" \
  --vendor "PUP BYOD Team" \
  --type deb \
  --input target/libs \
  --main-jar javafx-byod-client-1.0-SNAPSHOT.jar \
  --main-class com.pup.byod.javafxbyodclient.Launcher \
  --module-path target/libs \
  --module com.pup.byod.javafxbyodclient/com.pup.byod.javafxbyodclient.BYODApplication \
  --dest target/dist \
  --linux-shortcut \
  --linux-menu-group "Utility"
```

## 3. End-User Installation Guide

### 3.1 Windows Installation
1. Locate the generated `BYODClient-1.0.0.exe` installer.
2. Double-click to launch the installer.
3. *If prompted by Windows Defender SmartScreen (due to an unsigned executable), click **More Info** then **Run anyway**.*
4. Follow the setup wizard to choose an installation path (defaults to `C:\Program Files\BYODClient`).
5. Click **Finish**. A desktop shortcut and a Start Menu folder labeled **BYOD System** will be created.
6. Launch the app from the Desktop or Start Menu.

### 3.2 macOS Installation
1. Open the generated `BYODClient-1.0.0.dmg` file.
2. Drag the **BYODClient** icon into your **Applications** folder.
3. Open your Applications folder and launch the app.
4. *If blocked by Gatekeeper (unidentified developer error):*
   * Open **System Settings > Privacy & Security**.
   * Scroll down to find the blocked BYODClient notice and click **Open Anyway**.

### 3.3 Linux Installation
1. Install the `.deb` package using your package manager:
   ```bash
   sudo dpkg -i target/dist/byod-client_1.0.0_amd64.deb
   # If there are missing dependencies:
   sudo apt-get install -f
   ```
2. The application will be installed under `/opt/byod-client` and added to your desktop utilities search menu.
3. Run the app via the command line with:
   ```bash
   /opt/byod-client/bin/byod-client
   ```

## 4. Configuration & Troubleshooting

### 4.1 Modifying the Target Backend Endpoint
By default, the client points to the live Railway production backend:
`https://java-byod-backend-production.up.railway.app`

If the backend deployment server moves or needs to be pointed to a local server:
* Currently, the endpoint is set in `SessionManager.java`.
* To verify network connectivity, open a command prompt / terminal and run:
  ```bash
  curl -I https://java-byod-backend-production.up.railway.app/api/v1/health
  ```
  *(Make sure it returns `HTTP/1.1 200` or equivalent API health response).*

### 4.2 Common Issues and Solutions

#### Issue: WiX Toolset is not found during Windows packaging
* **Error**: `jpackage` returns an error saying WiX is required.
* **Solution**: Ensure WiX Toolset is installed and that the folder containing `candle.exe` and `light.exe` (usually `C:\Program Files (x86)\WiX Toolset v3.11\bin`) is in your system `PATH`. Restart your terminal/IDE and verify using `candle -?` in a new prompt.

#### Issue: Application launches but shows network connection errors
* **Error**: Alert dialog indicating a database or network connection failure.
* **Solution**: Verify that the client machine has internet access and can reach the backend domain. Check if local proxy/firewall configurations are blocking outgoing HTTPS requests to Railway.
